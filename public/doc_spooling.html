Spooling

Definición formal

El spooling (Simultaneous Peripheral Operation On-Line) es una especialización del buffering que utiliza el disco secundario como buffer de gran capacidad. Su propósito principal no es sólo compensar la diferencia de velocidades, sino lograr el desacoplamiento completo entre el proceso que genera los datos y el dispositivo que los consume — permitiendo que múltiples trabajos coexistan en cola mientras el dispositivo lento procesa cada uno en orden.

La diferencia clave con el buffering

Esta distinción es tan importante que merece una comparación directa:

Criterio Buffering Spooling
¿Qué desacopla? E/S y cómputo del mismo proceso Generación y consumo entre procesos distintos
¿Dónde almacena? RAM (volátil, nanosegundos) Disco (persistente, milisegundos)
¿Qué guarda? Fragmentos de un flujo de datos Trabajos completos (documentos, lotes)
Fallo de energía Pérdida total de datos Recuperación completa del trabajo
Multiprogramación Dentro de un proceso Entre múltiples procesos simultáneos

¿Por qué las impresoras necesitan spooling obligatoriamente?

La impresora encarna perfectamente el problema que resuelve el spooling. Tiene cuatro propiedades que la hacen un caso especial:

① Dispositivo exclusivo

Una impresora no puede intercalar bytes de dos documentos distintos — el resultado serían páginas mezcladas ilegibles. Solo puede atender un trabajo completo a la vez. El spooling garantiza atomicidad: cada trabajo se entrega entero.

② Velocidad abismal

Una impresora láser imprime 30-50 páginas/minuto ≈ 1-2 MB/s. La CPU genera datos a GB/s. Sin spooling, el proceso quedaría bloqueado durante minutos esperando que la impresora digiera el documento.

③ Múltiples usuarios

En entornos multiusuario (oficinas, universidades), varias personas quieren imprimir simultáneamente. El spooler serializa los trabajos en una cola FIFO: todos "terminan de imprimir" inmediatamente desde su perspectiva.

④ Persistencia necesaria

Si el sistema se reinicia a mitad de una impresión, el trabajo debe sobrevivir. Como el spool vive en disco, el spooler puede retomar exactamente donde dejó al reiniciarse.

PrintNightmare: cuando el spooler se convierte en vector de ataque

El spooler de Windows opera con privilegios de SYSTEM (el nivel más alto del sistema operativo) porque necesita acceder a hardware, instalar drivers en modo kernel y gestionar recursos privilegiados. Esta necesidad arquitectónica crea una superficie de ataque peligrosa.

🔴
CVE-2021-34527 — PrintNightmare

La vulnerabilidad surgió de un error en la función RpcAddPrinterDriverEx(): cualquier usuario autenticado del dominio podía invocarla para que el spooler (con privilegios SYSTEM) cargara un DLL malicioso desde una ruta de red controlada por el atacante. Resultado: ejecución de código arbitrario con los máximos privilegios del sistema.

Junio 2021

Investigadores de Sangfor descubren el zero-day y publican accidentalmente un PoC funcional en GitHub.

1 Julio 2021

Microsoft asigna CVE-2021-34527. El PoC ya se ha copiado en 34+ repositorios. Afecta a todas las versiones de Windows desde Win 7 hasta Win Server 2022.

6 Julio 2021

Microsoft publica parche de emergencia (out-of-band). Investigadores descubren que el parche solo cubre el componente RCE, no la escalada de privilegios local.

Post-2021

Microsoft introduce RestrictDriverInstallationToAdministrators, deshabilita la impresión remota entrante por defecto, y aísla procesos de renderizado. Desde 2021: cero nuevas vulnerabilidades RCE en el spooler — solo escaladas de privilegios menores.

La lección arquitectónica es profunda: cualquier servicio que combine alta privilegiación + superficie de ataque amplia + compatibilidad con código de terceros (drivers de fabricantes de impresoras) es estructuralmente riesgoso.
